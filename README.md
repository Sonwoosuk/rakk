<!-- 한두 줄로 프로젝트가 무엇인지, 왜 만들었는지 설명 -->

## ⚠️ 주의사항

<!-- 프로젝트 사용·열람 시 주의할 점 -->

- Firebase API 키는 환경 변수로 관리합니다. 직접 실행 시 `firebase-config.js`에 본인 키를 입력하세요.

## 📖 프로젝트 개요

<!-- 기획 배경, 목표, 타겟 사용자를 간략히 서술 -->

락고재는 100년 역사를 가진 전통 한옥 스테이입니다.  
기존 사이트의 노후화된 디자인을 개선하고, 브랜드 감성에 맞는 고급스러운 UI를 구현하는 것을 목표로 했습니다.

- **목표**: 전통과 현대의 조화를 담은 브랜드 사이트 리뉴얼
- **타겟**: 전통 문화 체험에 관심 있는 국내외 여행객
- **핵심 가치**: 고급스러움 · 전통 · 접근성

## 🔧 핵심 구현 내용

<!-- 기술적으로 공들인 부분을 구체적으로 작성 -->

**1. 자동 슬라이더 (히어로 섹션)**

```javascript
// 예시: 핵심 로직 발췌
// 실제 코드는 script.js 참고
function autoSlide() {
  currentIndex = (currentIndex + 1) % slides.length;
  goToSlide(currentIndex);
}
setInterval(autoSlide, 4000);
```

**2. 한/영 언어 전환**

- `data-ko` / `data-en` 속성으로 텍스트를 관리하고, `translate.js`에서 일괄 교체
- `localStorage`로 선택 언어 유지

**3. Firebase 예약 저장**

- Firestore 컬렉션 구조: `reservations/{uid}/bookings/{docId}`
- 로그인 상태에서만 저장 허용, 마이페이지에서 목록 조회

## 💡 프로젝트를 통해 배운 점

<!-- 기술적으로 새롭게 알게 된 점, 성장한 부분 -->

프로젝트를 개발하는 과정에서 제 환경에서는 정상적으로 동작하던 애니메이션이 일부 사용자 환경에서는 보이지 않는 문제가 발생했습니다. 
처음에는 브라우저 차이나 캐시 문제로 생각했지만, 
여러 환경에서 테스트하며 원인을 추적한 결과 전역으로 적용되어 있던 transition-duration: 0.001ms !important; 속성이 모든 transition 효과를 사실상 비활성화하고 있다는 점을 확인했습니다.

문제가 되는 전역 스타일을 하나씩 검토하고 테스트를 반복하며 원인을 찾아 수정했고, 
이후 사용자 환경에서도 애니메이션이 정상적으로 동작하는 것을 확인했습니다. 이 경험을 통해 개발 환경에서만 테스트하는 것에 의존하지 않고, 
다양한 브라우저와 기기에서 실제 동작을 검증하는 과정이 얼마나 중요한지 배울 수 있었습니다.

## 🪞 프로젝트 회고

<!-- 잘된 점, 아쉬운 점, 다음에 시도할 것 -->

**잘된 점**
### 잘된 점
- 브랜드의 전통적인 분위기와 현대적인 UI를 조화롭게 표현할 수 있었다.
- HTML, CSS, JavaScript만으로 인터랙션과 반응형 레이아웃을 구현하며 프론트엔드 구조를 직접 설계했다.
- 실제 서비스를 분석하여 UX 관점에서 개선점을 도출하고 디자인에 반영했다.
- Firebase를 활용해 로그인 기능을 구현하며 서비스 형태에 가까운 웹앱을 제작했다.

### 아쉬운 점
- 개발 과정에서 전역 CSS가 애니메이션에 영향을 주는 문제를 뒤늦게 발견해 디버깅에 많은 시간을 사용했다.
- 다양한 브라우저와 기기에서의 테스트를 프로젝트 후반에 진행해 수정 작업이 다소 늦어졌다.
- 초기 설계 단계에서 컴포넌트 구조와 스타일 관리 방식을 더 체계적으로 계획했다면 유지보수가 더 쉬웠을 것이라고 생각한다.

### 다음에 시도할 것
- 프로젝트 초반부터 크로스 브라우징 테스트를 병행하는 개발 프로세스를 적용할 예정이다.
- 컴포넌트 단위의 스타일 관리와 코드 구조를 더욱 체계화하여 유지보수성을 높일 계획이다.
- API 연동과 데이터 기반 기능을 추가해 실제 서비스 수준의 웹 애플리케이션으로 확장해 보고 싶다.
